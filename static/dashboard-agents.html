🧠 How the Engine Works

The G4H-RMA Quant Engine uses a 3-layer mathematical pipeline to find and execute pairs trading opportunities:

1. Kalman Filter

Dynamically tracks the hedge ratio (beta) between two correlated assets. Produces a pure z-score with zero look-ahead bias — telling you when the spread is statistically far from equilibrium.

2. EGARCH Volatility

Models asymmetric volatility — negative shocks increase future volatility more than positive ones. Classifies market regime as Low / Normal / Elevated / Crisis to adjust position sizing.

3. MCTS Decision

Monte Carlo Tree Search simulates 800+ future spread paths using Ornstein-Uhlenbeck mean-reversion. Outputs the expected value of going long, short, or holding the spread.

Kalman → EGARCH → MCTS → Risk Manager → Execute

🤖 How the 5 Trading Agents Work

Five specialized agents collaborate in a 10-second loop to scan, analyze, and execute trades:

🔍

ScoutAgent (scout-01)

Scans all configured pairs for Kalman z-score opportunities. Flags pairs where |Z| > 1.5 as potential trades.

📊

AnalystAgent (analyst-01)

Composite scoring: 50% MCTS expected value + 50% Kalman z-score. Ranks opportunities by confidence.

💹

TraderAgent (trader-01)

Determines execution timing and position sizing. Adjusts quantity based on current volatility regime.

🛡️

RiskAgent (risk-01)

Enforces daily trade limits ($100K max exposure, 10 trades/day). Halts trading during drawdown > 10% or crisis regime.

🚨

SentinelAgent (sentinel-01)

Black swan detection. Monitors for flash crashes, volatility spikes, and liquidity crises. Can halt all trading instantly.

Consensus Mechanism

A trade executes only when ≥2 agents agree with confidence > 60%. This prevents false signals and overtrading.

⚡ Complete Auto-Trading Setup Guide

1

Configure Broker Connection

Navigate to the 🔗 Connections tab. Find Alpaca (or your broker).

  • Toggle Enabled ON
  • Enter your API Key ID and Secret Key
  • Click Test — wait for "Connected" confirmation
  • Verify your account balance appears
2

Verify Pairs to Trade

The engine comes pre-configured with 15 equity pairs (AAPL/MSFT, NVDA/AMD, etc.).

  • Go to 🔍 Scanner tab to test pairs manually
  • Check z-scores and MCTS signals before enabling auto-trading
  • Pairs with |Z| > 2.0 are strongest opportunities
3

Understand Risk Limits

Default safety settings protect your capital:

  • Max daily trades: 10 (prevents overtrading)
  • Max position size: $25,000 per trade
  • Max portfolio exposure: $100,000 total
  • Max daily loss: $5,000
  • Max drawdown halt: 10%
  • Stop-loss: Z-score exit at ±5.0
4

Enable Auto-Trading

Go to the ⚡ Auto-Trading tab:

  • Click ▶ Start Engine — the agents begin their 10-second scan loop
  • Watch the Real-Time Agent Decisions feed on Dashboard tab
  • Toggle Auto-Trading ON — this enables live execution (paper by default)
  • Monitor active trades and agent statuses
5

Monitor & Review

Use these tabs to track performance:

  • 📊 Dashboard — Real-time agent decisions feed
  • 🤖 Agents — Individual agent metrics and win rates
  • 💰 P&L — Profit/loss summary and trade history
  • 🔍 Scanner — On-demand pair analysis
6

Emergency Stop

If anything goes wrong:

  • Go to ⚡ Auto-Trading tab
  • Click ⏹ Stop Engine — immediately halts all agent scanning
  • Toggle Auto-Trading OFF — prevents any new trades
  • Or run: ./stop-autodeploy.sh from terminal
